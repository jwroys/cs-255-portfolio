## Reflection

This portfolio submission highlights my work on the DriverPass system, which was designed as part of my system analysis and design course. The client, DriverPass, wanted a web-based system that allows customers to prepare for DMV driving tests through online training, practice exams, and scheduling real-world driving lessons. The goal of the system was to reduce the high failure rate of DMV tests by providing structured learning and hands-on experience in one centralized platform .

One area where I performed particularly well was translating the client’s needs into both functional and technical requirements. I clearly defined what the system should do, such as allowing users to schedule lessons, take practice tests, and manage accounts. I also ensured that administrative roles like the secretary and IT officer had the tools they needed to manage users and system operations. This helped create a system that is both user friendly and aligned with real business needs.

If I were to revise one part of my work, I would improve the UML diagrams, specifically the activity and sequence diagrams. While they accurately represent system behavior, they could be refined to improve clarity and readability. I would focus on simplifying flows and ensuring that each diagram communicates its purpose more efficiently, especially for stakeholders who may not have a technical background.

When interpreting the user’s needs, I focused heavily on the client interview and translated those needs into system features and requirements. For example, the need to stay current with DMV rules led to the integration requirement for external updates. Understanding the user’s needs is critical because the entire system depends on meeting those expectations. If the system does not align with how users actually operate, it will fail regardless of how well it is built.

My approach to designing software is structured and step by step. I start by identifying user needs and system requirements, then move into modeling using UML diagrams to visualize system behavior and structure. From there, I define technical requirements such as platform, security, and database needs. This approach ensures that the system is well planned before any development begins. In the future, I will continue using UML modeling, requirement analysis, and iterative refinement to improve system design and ensure that all components work together effectively.
